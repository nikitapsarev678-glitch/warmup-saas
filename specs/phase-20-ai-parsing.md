# Фаза 20 — AI Parsing (платный парсинг групп/аудитории)

> Читай SPEC.md перед началом. Фазы 10 и 16 должны быть выполнены (tokens, outreach).  
> Эта фаза добавляет платную фичу: поиск групп → парсинг участников/админов → пополнение leads.

## Цель
Как ты описал:
- ручной импорт остаётся бесплатным/базовым
- парсинг аудитории = **платная AI‑функция** (токены)

---

## Поток (MVP)

1) Пользователь задаёт запрос (ключевые слова, гео, лимиты).
2) Worker создаёт parsing‑job и списывает токены по факту результатов.
3) Runner:
   - ищет группы (поиск)
   - парсит админов (приоритет) и участников
   - сохраняет в `leads`
4) UI показывает прогресс и результат (сколько найдено/добавлено).

## Worker API

- `POST /parsing/start` (создать job, поставить в очередь)
- `GET /parsing/:id` (статус/прогресс)
- `GET /parsing/:id/result` (список лидов/метрики)

## Token metering

Списывать токены:
- за 1 добавленного лида (unit)
- + за AI‑классификацию ниши/группы (если включено)

При нуле токенов: job pause + уведомление (фаза 17).
